미니 다이어리 형태 간단하게 만들어 보기

*컨셉
유저가 다이어리를 제작, 유포하는 앱
다이어리는 삭제와 수정이 불가능.
다이어리의 내용물도 한번 작성되면 삭제와 수정이 불가능.
작성중인 또는 작성 완료된 다이어리는 다이어리별로 특정 유저와 공유 가능

* 사용될 프레임워크, 라이브러리 간단히
스프링 부트 사용. 우선 프론트랑 컨트롤러를 통해 서비스 형태가 갖춰지면 db추가

*작업시 유의사항
vue.js 쪽 숙련도가 매우 낮으니 높은 확률로 바퀴의 재발명 할 수 있음. 우선 원하는 내용을 쪼개서 이미 존재하는 컴포넌트인지 검색 및 gpt질의를 통해 존재하는지 확인한 후 없는 것만 제작할것. 
제작할 때 우선 gpt를 이용해 기초 틀을 뽑는게 숙련도가 0 인 상태에서 나오는 것보다는 좋은 형태일 수 있으니 gpt나 제미니를 이용해 우선 코드 뽑아볼것

* 작업할 내용과 순서

프론트
로그인 페이지 
- id,passwd 입력
- 로그인 버튼
(가입버튼은 나중에)

백엔드
어써링
- 유저의 로그인 요청 수용
- 접속권한 승인

현재 문제 - api 형식으로 클라이언트와 서버가 데이터를 주고 받는다면 권한에 대한 부분은 어떻게 설정하는가?
권한과 관련된 부분은 스프링 부트에서 세션을 관리해서 하는지, 아니면 다른 방법이 있는지 찾아보는게 좋을거같음. 

접속권한이 승인되면 이동할 유저페이지 제작. 유저페이지는 일단 공백으로 두기
